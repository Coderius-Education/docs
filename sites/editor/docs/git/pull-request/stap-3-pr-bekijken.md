# Stap 3: de pull request bekijken

Op de PR-pagina zie je drie tabbladen bovenaan:

## Conversation

Hier staan alle commentaren rondom deze PR, samen met de status (open, gemerged, gesloten). Helemaal onderaan kun je een nieuwe opmerking achterlaten. Voor jezelf is dit handig om "todo's" of vragen op te schrijven die je later wil aanpakken.

## Commits

Alle commits die in deze PR zitten. Klik op een commit om te zien wat er precies in veranderd is — handig als je later wilt navertellen waarom je iets gedaan hebt.

## Files changed

**Het belangrijkste tabblad.** Hier zie je de **diff**: links de oude versie, rechts de nieuwe (of in één kolom met `+` en `-`-regels). Groen = toegevoegd, rood = verwijderd.

### Commentaar op een specifieke regel

1. Zweef met je muis over een regel in de diff
2. Er verschijnt een blauw **`+`**-icoon links van de regel
3. Klik erop → tekstveld
4. Typ je commentaar en klik **Add single comment** (of **Start a review** als je meer comments wil bundelen)

Dit is *de* manier waarop teams code reviewen. Ook voor jezelf is het waardevol om hier kritisch naar je eigen diff te kijken: zie ik nog typo's? Heb ik commentaar laten staan? Vergat ik een bestand?
